

# Library Management Database

A relational database design for a Library Management System implemented using SQL.

## Features

- Relational schema design
- Primary & Foreign keys
- Table relationships
- Join queries
- Aggregation queries

## Database Structure

Tables Included:

- Authors
- Books
- Members
- Loans

## Key Concepts Demonstrated

- One-to-Many relationships
- Foreign key constraints
- INNER JOIN
- LEFT JOIN
- GROUP BY
- Aggregation functions (COUNT)

## Files

- schema.sql → Table creation and constraints
- queries.sql → Sample queries and joins

## Example Queries

- Retrieve all borrowed books
- Count number of books per author
- List active loans

## What This Project Demonstrates

- Database normalization principles
- Referential integrity
- Practical SQL querying
- Relational modeling

## Future Improvements

- Add triggers for automatic return date updates
- Add stored procedures
